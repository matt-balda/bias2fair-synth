# Experimental Scenarios: Synthetic Data and Fairness

## 1. Objective
Evaluate the effectiveness of synthetic data generation as a strategy for bias mitigation in supervised machine learning, and assess its impact on the trade-off between predictive performance and model fairness under controlled and explicitly defined scenarios.

---

## 2. Inputs
- **Dataset:** COMPAS (`data/compas-scores-two-years.csv`), filtered per ProPublica guidelines
- **Target:** `two_year_recid` (binary: 0 = no recidivism, 1 = recidivated within 2 years)
- **Sensitive Attribute:** `race` (binary encoded: 0 = African-American [Unprivileged], 1 = Caucasian [Privileged])
- **Other encodings applied in `data_loader.py`:**
  - `sex`: Male=1, Female=0
  - `c_charge_degree`: F=1, M=0
  - `age_cat`: one-hot encoded via `pd.get_dummies` (drop_first=True)
  - Rows retained: only African-American and Caucasian races

---

## 3. Fixed Models
- Logistic Regression (`random_state=seed`, `max_iter=1000`)
- Random Forest (`random_state=seed`, `max_depth=7`)
- CatBoost (`random_state=seed`, `iterations=500`, `task_type='GPU'`, `verbose=0`)

> **Note:** All models receive `StandardScaler`-transformed features (fitted on train, applied to test).
> The scaler is applied to all numeric columns (`int64`, `int32`, `float64`, `float32`, `uint8`).
> This is applied **after** final training set construction and **before** `model.fit()`.

## 5. Experimental Scenarios (STRICT DEFINITIONS)

### Scenario S1 — Baseline (No Intervention)
- No bias mitigation
- No synthetic data
- Train on original dataset

---

### Scenario S2 — Bias Mitigation Only
- Apply **Reweighing** (AIF360 `sklearn.preprocessing.Reweighing`) on training set
- No synthetic data
- Models trained with `sample_weight` vector from Reweighing
- No modification to test data

---

### Scenario S3 — Synthetic Data (Full Augmentation)
- No bias mitigation
- Generator fitted on **entire training set**
- `synth = gen.sample(len(train))`
- Final training set = real + synthetic (doubled size, same overall distribution)

---

### Scenario S4 — Synthetic Data (Minority Group Only)
- No bias mitigation
- Generator fitted **exclusively on the minority subset** (`train[race == 0]`)
- `synth = gen.sample(len(minority))`
- Final training set = real + synthetic (minority-only oversampling)
- Rationale: fitting on minority ensures generated data reflects minority distribution

---

### Scenario S5 — Combined (Bias Mitigation + Synthetic Data)
- Reweighing applied first → compute `weights`
- Generator fitted on **resampled-with-replacement** training set using `weights` as probabilities:
  `train_balanced = train.sample(n=len(train), replace=True, weights=weights, random_state=seed)`
- `synth = gen.sample(len(train))`
- Final training set = real + synthetic
- Reweighing recomputed on expanded `train_final` → new `weights` applied at `model.fit()`
- Rationale: since generators don't accept `sample_weight`, empirical resampling transfers the reweighing signal into the generator's fit distribution

#### Fairness (Signed Definitions — Unprivileged minus Privileged)
All fairness metrics use signed formulas to preserve directionality:
- **Statistical Parity Difference (SPD):** `Pr(Ŷ=1 | race=0) − Pr(Ŷ=1 | race=1)` → negative = bias against Afr.-Am.
- **Disparate Impact (DI):** `Pr(Ŷ=1 | race=0) / Pr(Ŷ=1 | race=1)` → <1 = bias against Afr.-Am., >1 = overcorrection
- **Equal Opportunity Difference (EOD):** `TPR(race=0) − TPR(race=1)` → signed, measures recall parity
- **Average Absolute Odds Difference (AAOD):** via `fairlearn.equalized_odds_difference` (absolute)
